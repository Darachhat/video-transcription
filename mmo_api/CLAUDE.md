# AI Interaction Guidelines

## User Profile
Tailor all technical explanations and architectural suggestions for a 4th-year Information Technology Engineering student currently working as a Software Engineer. The user has a strong background in backend development, complex API integrations, and database management. 

## Communication Style
* Skip basic programming concepts and tutorial-style explanations unless explicitly requested.
* Focus on production-ready, modular architecture.
* Keep responses concise and prioritize writing clean, functional code.
* Highlight any potential rate-limiting, compute costs, or optimization bottlenecks immediately.

## Coding Standards
* Language: Python 3.10+.
* Use explicit type hinting for all function parameters and return types.
* Implement robust error handling (e.g., `try/except` blocks for network requests or file missing errors).
* Use environment variables (`.env`) for all API keys and configuration settings.
* Include concise docstrings for core pipeline functions.
